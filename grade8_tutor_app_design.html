01 · Overview

Overview & Goals

A focused revision platform that mirrors the school curriculum for a Grade 8 student. Each study session moves through structured stages — from passive content consumption to active recall — building durable understanding rather than surface familiarity.

Core Design Principle Every session follows the same spine: introduce → explore → recall → test → reflect. Materials are uploaded by the admin and organised by subject and topic. Students work through stages at their own pace; progress is tracked and visible on a dashboard.

The platform is intentionally small — one admin (parent or guardian), two students, and one test account — keeping it lightweight, private, and easy to manage without a backend team. All uploaded materials (slides, podcasts, guides, mind maps, infographics) live in a structured content library organised by subject and topic unit. Questions — multiple choice and flashcards — are attached to topics and surfaced during the practice stages of each session.

03 · Learning Materials

Learning Materials

Five content types can be uploaded per topic. The admin uploads source files; the platform stores and serves them to students in the appropriate viewer.

🖥️
Slideshows
PowerPoint or PDF presentation slides exported from school lessons. Displayed in a built-in slide viewer with forward/back navigation and fullscreen mode.
Formats: .pptx · .pdf · .key
🎧
Podcasts / Audio
Recorded lesson summaries, teacher narrations, or curated educational podcasts. Played with a simple audio player supporting speed control (0.75×–2×) and timestamps.
Formats: .mp3 · .m4a · .wav
📋
Study Guide Q&A
Structured question-and-answer documents covering key concepts. Displayed as a collapsible accordion — student reads the question, then reveals the answer to self-check.
Formats: .pdf · .docx · structured text

Visual concept maps showing relationships between ideas. Rendered as zoomable, pannable images. Can be exported from any mind-mapping tool and uploaded as an image.
Formats: .png · .svg · .pdf
📊
Infographics
Visual summaries of topics — timelines, diagrams, process charts, comparison tables. Displayed in a full-width image viewer with optional pinch-to-zoom on mobile.

04 · Study Sessions

Study Session Flow

Every topic follows a five-stage session structure. Stages are sequential — completing one unlocks the next. Students cannot skip ahead but may revisit earlier stages at any time.

1
🎬
Introduce
Watch the slideshow and listen to the podcast. Passive intake of new material from school lessons.
~15 min
2
🗺️
Explore
Study the mind map and work through the Study Guide Q&A with answers hidden. Build the concept structure.
~10 min
3
🧠
Recall
Flashcard practice. Cards are shown front-first; student self-rates recall as Easy, Medium or Hard, feeding the spaced-repetition queue.
~10 min
4
✅
Test
Multiple-choice quiz with 10–20 questions. Randomised order. Immediate feedback after each answer. Score recorded.
~10 min
5
📊
Review
Session summary: quiz score, flashcard performance, time spent, and suggested next topic or retry recommendation.
~3 min
Spaced Repetition Logic Flashcards rated "Hard" are shown again within the same session. Cards rated "Medium" reappear in the next session on that topic. Cards rated "Easy" are scheduled further out. This is managed per-student and per-card.

The admin can configure which materials are active for each stage and whether stages require a minimum score to unlock the next (e.g. must score ≥ 60% on the quiz before Stage 5 is available). This gate is optional and toggled per subject.

05 · Assessment

Assessment Types

Two question formats are used: multiple-choice quizzes for scored testing and flashcards for self-directed recall practice.

Feature Multiple Choice Quiz Flashcards
Session stage Stage 4 — Test Stage 3 — Recall
Question format Question + 4 answer options (one correct) Front (prompt) + Back (answer/explanation)
Feedback Immediate — correct/incorrect + explanation shown Student self-rates: Easy / Medium / Hard
Score % correct, stored per attempt No numeric score; affects spaced repetition schedule
Question count 10–20 per topic (admin-set) All cards in deck; repeats for "Hard" cards
Randomisation Question order and answer order randomised each attempt Card order randomised each session
Admin authoring Question, 4 options, correct answer, optional explanation Front text, back text, optional image on either side
Image support Optional image in question stem Optional image on front or back
Retry Unlimited retries; all attempts tracked Continuous practice; no retry concept

10 · Tech Stack

Tech Stack

Recommended technology choices for a lightweight, self-hostable build. Designed to be run on a single low-cost server or deployed as a managed app.

Frontend
Next.js (React)
App Router, server components, TypeScript. Simple responsive UI with Tailwind CSS. Mobile-first — student will use on tablet or phone.
Backend / API
Next.js API Routes
Or a small Express/Fastify server. REST API for all data operations. No complex microservices needed for a 4-user app.
Database
PostgreSQL
Supabase (managed, free tier) or self-hosted. Simple schema matching the data model above. Prisma ORM recommended.
File Storage
Supabase Storage or S3
Stores uploaded slides, audio, images. Signed URLs for secure delivery. Max file size: 100 MB per upload.
Authentication
NextAuth.js
Credentials provider (email + password). Session stored in JWT. 4 user accounts, no OAuth needed. Role stored in session token.
Hosting
Vercel + Supabase
Free tier sufficient for 4 users. Alternatively: Railway, Render, or a $5/month VPS (DigitalOcean Droplet).
No-Code Alternative If development resources are limited, the same structure can be built in Notion (content library) + Typeform (quizzes) + Airtable (progress tracking). This reduces functionality but allows rapid deployment without code.

12 · UX Principles

UX Principles

Design decisions guided by the specific context: a Grade 8 student using the app for after-school revision, likely on a tablet or phone.

Principle Implementation
Low friction start Landing on the student home shows today's suggested topic (based on what's overdue or not started). One tap to begin a session — no hunting through menus.
Progress is always visible Each topic shows a small progress bar (stages 1–5) and a star count. The student always knows where they are and what comes next.
Positive reinforcement Completing a stage shows a brief celebration animation. Quiz score improvement over last attempt is highlighted. Streak counter is prominent.
Mobile-first layout All content viewers work on small screens. Slide viewer swipes left/right. Audio player has large touch targets. Flashcards flip on tap.
No dead ends If a topic has no materials yet, it shows "Coming soon" rather than an error. If a quiz has no questions, Stage 4 is hidden until questions are added.
Admin efficiency Bulk upload for materials. Copy questions between topics. Duplicate a topic template. All admin actions accessible from a sidebar, not buried in settings.
Distraction-free study mode Entering a session hides navigation. Only the current stage content and a minimal progress indicator are shown. Exit button is present but not prominent.
Consistent typography All question text, material titles, and labels use the same readable font at 16px minimum. No small print. High contrast for readability.
